# Sphinx University — تحديثات المشروع

## 📂 الملفات الموجودة هنا

| الملف | الحالة |
|---|---|
| `login.html` | معدّل — ميزات أمان حقيقية أضيفت |
| `index.html` | معدّل — تصحيح خطأ برمجي كان موجود في الملف الأصلي |
| `admin.html` | معدّل بالكامل — نظام دخول حقيقي مرتبط بـ login.html |
| `supabase_setup.sql` | **جديد — يجب تشغيله** في Supabase SQL Editor |
| `edge-function-create-user.ts` | **جديد (اختياري)** — لتفعيل "إضافة مستخدم" من لوحة الأدمن بأمان |

---

## ⚠️ خطوة أولى ضرورية: شغّل ملف SQL

اذهب إلى لوحة Supabase → **SQL Editor** → افتح ملف جديد → الصق محتوى
`supabase_setup.sql` بالكامل → Run.

هذا الملف:
- ينشئ جدول `users` (لو لم يكن موجوداً).
- ينشئ جدولين جديدين: `admin_activity_log` و `login_attempts`.
- **يفعّل Row Level Security (RLS)** — هذا هو الجزء الأهم. بدونه، أي
  شخص يفتح "أدوات المطور" في المتصفح يقدر يغيّر بياناته الشخصية
  ليصبح admin بنفسه، حتى لو واجهة admin.html كانت تمنعه ظاهرياً.

---

## ⚠️ خطوة ثانية: عيّن نفسك كأول admin

بعد تشغيل ملف SQL، وبعد إنشاء حسابك من `login.html` (Sign Up)، شغّل
هذا السطر **مرة واحدة فقط** في SQL Editor:

```sql
update public.users set is_admin = true where phone = '01066781530';
```

(غيّر رقم التليفون لو عايز ده يكون رقم تليفون حسابك الحقيقي اللي
عملت بيه Sign Up).

بعد كذا، تقدر تفتح `admin.html` مباشرة وهتدخل تلقائي لأن نفس جلسة
تسجيل الدخول بتاعتك في `login.html` هي اللي بتتحقق منها admin.html.

---

## 🔐 ليه شلت الباسورد المكشوف من admin.html؟

الكود القديم كان بيتحقق من رقم التليفون وكلمة السر **جوه المتصفح
نفسه** (JavaScript). ده معناه:
- أي حد يفتح "View Page Source" أو Developer Tools يقدر يقرا
  كلمة السر مباشرة.
- أي حد يقدر يستدعي قاعدة البيانات مباشرة من الكونسول (`fetch`)
  ويتجاوز شاشة الدخول بالكامل، من غير ما يحتاج الباسورد أساساً.

دلوقتي admin.html بيسأل قاعدة البيانات نفسها "هل الشخص اللي عامل
session دلوقتي هو admin؟" وقاعدة البيانات (مش الصفحة) هي اللي
بتحدد الإجابة، محمية بـ RLS مينفعش يتم التلاعب فيها من المتصفح.

---

## 🚫 ميزة "Add User" في admin.html معطّلة مؤقتاً — وده مقصود

لو فتحت قسم "Add User" هتلاقي رسالة توضيحية. السبب:

إنشاء حساب نيابة عن شخص تاني من المتصفح مباشرة (زي ما كان الكود
القديم بيعمل بـ `sb.auth.signUp()`) كان بيعمل حاجة خطيرة: بيسجل
خروج الـ admin من حسابه ويسجله دخول كالمستخدم الجديد تلقائياً!

الحل الصحيح محتاج **سيرفر** (Supabase Edge Function) يستخدم
الـ Service Role Key بأمان من غير ما يظهر في أي صفحة HTML. كود
الفنكشن جاهز في `edge-function-create-user.ts` مع تعليمات النشر
في أول الملف.

**لحد ما تنشر الفنكشن:** أسهل حل هو إن المستخدمين الجدد يسجلوا
بنفسهم من `login.html` (Sign Up) وبعدين انت كـ admin تقدر تغيّر
صلاحيتهم (User/VIP/Admin) من جدول "Manage Users" في admin.html.

## ✅ ميزات حقيقية تمت إضافتها في login.html

- **قفل الحساب بعد محاولات فاشلة متكررة** (5 محاولات / 15 دقيقة).
- **تسجيل محاولات الدخول** في جدول `login_attempts` (يقدر admin
  يشوفه بس).
- **صفحة "تعيين كلمة سر جديدة"** فعلياً تظهر لما المستخدم يضغط على
  رابط "Forgot Password" من الإيميل (كان ناقص قبل كده).
- **تنبيه عند الدخول من جهاز/متصفح جديد**.
- **احترام حالة "تعطيل الحساب"** (`is_disabled`) اللي يقدر admin
  يفعّلها — المستخدم المعطّل مينفعش يسجل دخول حتى لو كلمة السر
  صحيحة.

---

## 📝 ملاحظة عن "2FA / WebAuthn / بصمة الإصبع"

المطلوبة في القائمة الأصلية، دي محتاجة سيرفر backend حقيقي مش بس
صفحة HTML — مينفعش تتعمل بأمان بمفتاح Supabase العام (publishable
key) لوحده. لو عايز ده فعلياً، الخطوة التالية هي عمل Edge Functions
إضافية زي اللي عملتها لإنشاء المستخدمين، وده موضوع منفصل لو حابب
نكمل فيه بعدين.
