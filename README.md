# Sphinx University

منصة تعليمية لكلية الحاسبات والذكاء الاصطناعي — جامعة سفينكس، أسيوط.
HTML/CSS/JS بالكامل (بدون أي backend) + Supabase كقاعدة بيانات ونظام مصادقة.

---

## 🚨 خطوة واحدة إلزامية قبل أي تشغيل: شغّل ملف SQL

بدون هذه الخطوة، **تسجيل الدخول والتسجيل الجديد لن يعملا** — مش
مجرد مشكلة أمان، المشكلة دلوقتي أساسية في عمل النظام نفسه.

### السبب (مهم تفهمه قبل ما تبدأ):

1. فعّلنا **Row Level Security (RLS)** على جدول `users`. ده معناه:
   حد مايقدر يقرا صف حد تاني في الجدول إلا لو كان admin. ده صحيح
   ومطلوب أمنياً، لكنه يكسر حاجتين كانت الصفحة بتعتمد عليهم:
   - تسجيل الدخول بالتليفون أو الرقم الجامعي (محتاج يلاقي الإيميل
     المرتبط بيهم *قبل* ما يكون فيه تسجيل دخول أصلاً).
   - فحص "هل الرقم ده مسجل قبل كده؟" عند التسجيل الجديد.

2. لو إيميل التفعيل (Email Confirmation) شغال في مشروعك على
   Supabase، فإن `signUp()` ميرجعش جلسة فعّالة فوراً — وأي محاولة
   من المتصفح للكتابة في جدول `users` بعد التسجيل مباشرة هتفشل.

**الحل لكل ده موجود في `supabase_setup.sql`** — فيه دوال خاصة
(RPC functions) تحل المشكلة الأولى، و **trigger** على قاعدة
البيانات نفسها يحل المشكلة الثانية تلقائياً.

### طريقة التشغيل:

1. افتح لوحة Supabase بتاعتك → **SQL Editor**.
2. افتح ملف جديد (New query).
3. الصق **كل** محتوى `supabase_setup.sql` (من أول سطر لآخر سطر).
4. دوس **Run**.
5. تأكد إنه نفّذ بدون أخطاء (لو ظهر خطأ، ابعته لي بالكامل).

## 👑 خطوة تانية: خلي نفسك Admin

بعد تشغيل SQL، روح اعمل حساب عادي من `login.html` (Sign Up) برقم
تليفونك الحقيقي. بعد كذا، رجع لـ SQL Editor وشغّل سطر واحد بس:

```sql
update public.users set is_admin = true where phone = '01066781530';
```

(غيّر الرقم لو سجلت برقم مختلف). بعد كذا `admin.html` هيفتح تلقائياً
لما تكون مسجل دخول بنفس الحساب ده — مفيش باسورد تاني منفصل.

## 📦 رفع المشروع على GitHub

```bash
git init
git add .
git commit -m "Initial commit — Sphinx University platform"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO.git
git push -u origin main
```

المفتاح الموجود في الكود (`sb_publishable_...`) هو **مفتاح عام
آمن للنشر** (Publishable/Anon key) — مصمم خصيصاً ليكون مرئياً في
كود الواجهة الأمامية ومحمي بـ RLS، فمش هناك مشكلة في رفعه على
GitHub أو نشره من خلال GitHub Pages/Netlify/Vercel.

**لكن لو شغّلت مشروع الـ Edge Function (`edge-function-create-user.ts`)
يوماً ما، ده محتاج الـ Service Role Key — وده لازم يبقى env variable
على سيرفر Supabase نفسه، مش في أي ملف هترفعه على GitHub أبداً.**

## 📂 الملفات

| الملف | الوظيفة |
|---|---|
| `index.html` | الصفحة الرئيسية / لوحة تحكم الطالب |
| `login.html` | تسجيل الدخول / إنشاء حساب / دخول كضيف |
| `admin.html` | لوحة تحكم الأدمن (نفس جلسة login.html) |
| `supabase_setup.sql` | **شغّله أول حاجة** — الجداول + RLS + الدوال |
| `edge-function-create-user.ts` | اختياري — لتفعيل "إضافة مستخدم" من الأدمن لاحقاً |

## 🔧 إصلاحات حقيقية تمت في هذا التحديث

### مشاكل كانت بتمنع تسجيل الدخول/التسجيل من الأساس:
- ❌ كان البحث عن مستخدم بالتليفون/الرقم الجامعي قبل تسجيل
  الدخول بيرجع فاضي دايماً بسبب RLS → ✅ بقى عبر دالة RPC آمنة.
- ❌ فحص تكرار البيانات عند التسجيل كان بيفشل بصمت → ✅ دالة RPC
  منفصلة.
- ❌ إدراج صف المستخدم الجديد بعد `signUp()` كان بيفشل لو تأكيد
  الإيميل مفعّل → ✅ trigger في قاعدة البيانات يعمل ده تلقائياً.
- ❌ حسابات Google كانت كلها بترث `university_id` ثابت وده يتعارض
  مع قيد uniqueness بعد أول حساب → ✅ تم تصحيحه.
- ❌ تسجيل دخول الضيوف كان بيحاول يقرا حساب ضيف قديم بطريقة
  مستحيلة تقنياً (لا يوجد باسورد للضيوف) → ✅ تم تبسيط وتصحيح
  المنطق بالكامل.

### مشاكل أمان:
- ❌ باسورد admin.html كان مكتوب صراحة في الكود (أي حد يقدر يقراه
  من View Source) → ✅ بقى معتمد على نفس جلسة Supabase الحقيقية.
- ❌ كان فيه طريقة لتجاوز شاشة دخول الأدمن بسطر واحد في الكونسول
  (`localStorage.setItem('sphinx_admin_logged','1')`) → ✅ تمت
  إزالتها بالكامل.

## ⚠️ ميزة معطّلة عمداً: "Add User" من لوحة الأدمن

إنشاء حساب نيابة عن شخص تاني من المتصفح مباشرة كان بيعمل حاجة
خطيرة: بيسجل خروج الأدمن من حسابه ويسجله دخول كالمستخدم الجديد!
الحل الصحيح محتاج سيرفر (Edge Function) — الكود جاهز في
`edge-function-create-user.ts` مع تعليمات النشر، بس النشر ده خطوة
منفصلة لو حبيت تكملها بعدين. **لحد ما تنشرها، الطلاب الجدد
يسجلوا بنفسهم من `login.html`** وانت تغيّر صلاحياتهم بعد كده من
لوحة الأدمن.
